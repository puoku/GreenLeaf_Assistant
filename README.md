# GreenLeaf Assistant

Telegram-бот для магазина.

Что умеет сейчас:
- отвечает на FAQ: адрес, график, доставка, оплата, гарантия, возврат, самовывоз, контакты
- ищет товары по каталогу
- показывает цену и остаток товара
- принимает заказ в личке и в клиентском чате Telegram
- принимает бронь
- умеет принять бронь из обычного сообщения со списком товаров
- если части товаров нет, предлагает оформить бронь без них
- отправляет клиенту подтверждение заказа или брони в личные сообщения
- даёт клиенту кнопки: подтвердить, отклонить, нужен менеджер
- не списывает товар автоматически после сообщения клиента
- списывает товар только после подтверждения заказа или брони в веб-админке
- позволяет менеджеру подтверждать и отклонять заказы и брони в админке
- удаляет подозрительные ссылки от не-админов
- открывает простую веб-админку

Проект находится в папке `greenleaf_bot_project_v2`.

Как работает сейчас:
- клиент пишет в личку боту или в клиентский чат заказов
- бот собирает заказ или бронь
- если сообщение пришло из общего чата, подтверждение заявки бот отправляет клиенту в личку
- клиент подтверждает только правильность собранной заявки
- наличие товара и итоговое подтверждение проверяются менеджером в админке
- остатки товара меняются только после действия менеджера в админке

## Запуск

```bash
cd greenleaf_bot_project_v2
cp .env.example .env
python3.12 -m venv .venv
source .venv/bin/activate
python -m pip install --upgrade pip
pip install -r requirements.txt
python -m app.main
```

После запуска:
- бот работает в Telegram
- админка доступна на `http://localhost:8000/admin`
- проверка сервиса: `http://localhost:8000/health`

## Что заполнить в `.env`

Минимально:

```env
BOT_TOKEN=your_bot_token
BOT_USERNAME=GreenLeafBot
OPENAI_API_KEY=your_openrouter_api_key
OPENAI_MODEL=openai/gpt-4o-mini
ADMIN_USERNAME=admin
ADMIN_PASSWORD=change-me
```

Дополнительно:

```env
MANAGER_CHAT_ID=0
MANAGER_TOPIC_ID=
```

Если менеджерские действия выполняются только в админке, `MANAGER_CHAT_ID` можно оставить `0`.
