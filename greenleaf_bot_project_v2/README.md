# GreenLeaf Assistant

Готовый MVP для Telegram-магазина:
- бот для лички, групп и супергрупп с топиками;
- поиск по каталогу из БД;
- FAQ-ответы;
- оформление заказа и брони;
- уведомления менеджеру в отдельный чат;
- удаление подозрительных ссылок от не-админов;
- простая веб-админка.

## Что уже реализовано

### Бот
- отвечает на FAQ: адрес, график, доставка, оплата, гарантия, возврат, самовывоз, контакты;
- ищет товары по названию, синонимам и близким совпадениям;
- показывает найденный товар с ценой, остатком и кнопками;
- принимает заказ в несколько шагов;
- принимает бронь в несколько шагов;
- переводит клиента на менеджера, если бот не уверен;
- работает в личке и в супергруппах, включая топики.

### Менеджерский чат
- каждый новый заказ отправляется в отдельный чат менеджеров;
- у заказа есть кнопки: подтвердить / в работу / отменить / нужен менеджер;
- каждая новая бронь тоже уходит в менеджерский чат;
- менеджер подтверждает бронь одной кнопкой.

### Админка
- HTTP Basic авторизация;
- просмотр последних заказов и броней;
- добавление товаров;
- добавление FAQ;
- просмотр списков заказов и броней.

## Стек
- Python 3.12
- aiogram 3
- FastAPI
- SQLAlchemy 2
- SQLite или PostgreSQL
- OpenRouter через OpenAI-compatible API для классификации сообщений
- RapidFuzz для быстрого fuzzy-поиска

## Быстрый запуск локально

### 1. Скопируй переменные окружения

```bash
cp .env.example .env
```

### 2. Заполни `.env`

Минимально нужно указать:

```env
BOT_TOKEN=токен_от_BotFather
BOT_USERNAME=GreenLeafBot
DATABASE_URL=sqlite+aiosqlite:///./greenleaf.db
OPENAI_API_KEY=твой_openrouter_api_key
ADMIN_USERNAME=admin
ADMIN_PASSWORD=сильный_пароль
MANAGER_CHAT_ID=-100xxxxxxxxxx
```

### 3. Установи зависимости

```bash
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

### 4. Запусти проект

```bash
python -m app.main
```

После запуска будет доступно:
- бот — в Telegram;
- админка — `http://localhost:8000/admin`;
- healthcheck — `http://localhost:8000/health`.

## Как подключить бота в Telegram

### 1. Создай бота через BotFather
Получишь токен и username.

### 2. Добавь бота в супергруппу
Выдай ему права администратора:
- удаление сообщений;
- чтение сообщений;
- при необходимости управление топиками.

### 3. Добавь бота в чат менеджеров
Потом узнай `chat_id` этого чата и вставь в `.env` как `MANAGER_CHAT_ID`.

Самый простой способ узнать `chat_id`:
- временно запусти бота;
- напиши что-нибудь в менеджерский чат;
- добавь временный лог `print(message.chat.id)` в обработчик;
- перезапусти и посмотри id в консоли.

## Стартовые данные
При первом запуске автоматически создаются:
- товары из твоего списка;
- базовые FAQ из твоего ТЗ.

## Как работает ценообразование
В коде заложено:
- `price_partner` — партнёрская цена;
- `price_regular` = `price_partner * PARTNER_PRICE_MULTIPLIER`.

По умолчанию множитель = `2`.

## Как работает ИИ
Сейчас схема гибридная:
- сначала FAQ и быстрый поиск по каталогу;
- если правило не сработало — OpenRouter пытается классифицировать запрос;
- если уверенности нет — бот передаёт клиента менеджеру.

Это даёт быстрые ответы на частые вопросы и не ломает сценарий, если вопрос задан свободно.

## Где менять логику
- `app/bot/handlers.py` — все Telegram-сценарии;
- `app/services/product_search.py` — поиск по товарам;
- `app/services/faq.py` — подбор FAQ;
- `app/services/llm.py` — интеграция с OpenRouter;
- `app/services/link_moderation.py` — подозрительные ссылки;
- `app/web_admin.py` — веб-админка.

## Важные замечания
1. Сейчас бот отправляет **новые** заказы и брони менеджеру сразу, а статус уже меняется менеджерскими кнопками.
2. Антиспам сделан как **эвристика подозрительных ссылок**, потому что ты попросил удалять не всё подряд, а только подозрительное.
3. Для production лучше использовать PostgreSQL, Nginx и webhook.
4. Если `OPENAI_API_KEY` не задан, бот всё равно работает, но только на правилах и fuzzy-поиске.
